## 一、项目概述

一款 Whack-a-Mole 变体反应力休闲游戏。宝宝躺在床上准备入睡，玩家需要及时处理房间内随机出现的干扰事件，防止宝宝被吵醒。

### 本轮迭代范围

- 只做第一关「第一夜·初识」
- 两个事件：手机响铃（长按2秒）、闹钟响了（点击）
- 简易主菜单 → 关卡 → 结算面板
- 占位图美术（纯色块 + 文字标签）
- 数据驱动框架，为后续14关预留扩展能力

### 不包含

- 关卡选择界面、星级评价、存档系统
- 拖拽 / 组合交互类型（后续关卡引入）
- 梦境场景
- 音频系统
- 帧动画 / 粒子特效

## 二、核心玩法机制

### 游戏循环

```
主菜单 [开始] → 关卡开始（安睡度100，倒计时60秒）
    │
    ├─ 每隔8秒从事件池随机抽一个事件生成
    │   事件池：phone_ring / alarm_clock
    │   同时最多1个事件存在
    │
    ├─ 事件出现后，屏幕上显示对应的交互目标
    │   ├─ 玩家在时间窗口内操作成功 → 安睡度不变，事件消失
    │   └─ 玩家超时未处理 → 安睡度 -15，事件消失
    │
    ├─ 安睡度 <= 0 → 宝宝哭醒 → 失败面板（重试 / 回主菜单）
    └─ 倒计时归零且安睡度 > 0 → 通关 → 成功面板（重玩 / 回主菜单）
```

### 事件定义

| 字段 | phone_ring 手机响铃 | alarm_clock 闹钟响了 |
|------|---------------------|----------------------|
| 交互类型 | 长按 | 点击 |
| 操作说明 | 按住手机节点2秒 | 快速点击闹钟节点 |
| 时间窗口 | 5秒 | 4秒 |
| 超时惩罚 | -15 | -15 |
| 成功惩罚 | 0 | 0 |
| 视觉占位 | 蓝色矩形 + "📱长按" | 红色矩形 + "⏰点击" |

### 长按交互细节

- 手指按住后显示进度条（ProgressBar），2秒填满即成功
- 手指中途松开，进度条重置，需要重新按住
- 时间窗口是事件整体存活时间（5秒），在此期间可以多次尝试

### 安睡度显示

屏幕顶部进度条，从绿色（100）渐变到红色（0）。

### 宝宝状态反馈

| 安睡度范围 | 显示 |
|-----------|------|
| > 50 | 😴 |
| 20 - 50 | 😟 |
| 1 - 19 | 😫 |
| = 0 | 😭（触发失败） |

### 1. 主菜单（MainMenu.scene）

- 全屏深蓝纯色背景
- 居中标题文字 "Baby Sleep"
- 标题下方一个「开始」按钮
- 点击后切换到 RoomLevel.scene

### 2. 关卡场景（RoomLevel.scene）

- **背景层**：全屏浅米色矩形，代表房间
- **宝宝层**：屏幕下半部居中，矩形色块代表床 + 上方文字表情代表宝宝
- **HUD 层**（顶部固定）：
  - 左侧：安睡度进度条（带数字）
  - 右侧：倒计时文字 MM:SS
- **事件层**：事件节点在预设固定位置出现
  - 手机：房间右侧（床头柜位置）
  - 闹钟：房间左侧（桌子位置）
  - 事件节点包含：色块背景 + 操作提示文字 + 时间窗口倒计时条
  - 长按类事件额外显示：操作进度条

### 3. 结算面板（Prefab 弹窗）

- 半透明黑色遮罩
- 居中白色面板
- 标题文字：成功 "通关！" / 失败 "宝宝哭醒了..."
- 显示最终安睡度数值
- 两个按钮横排：「重玩」「回主菜单」

### 事件出现位置

不做随机位置，每种事件有一个固定出现锚点。占位图阶段玩家能建立空间记忆，后续替换为房间内实际物品位置。

### 核心模块职责与通信

```
GameManager（静态类单例，不挂载节点）
    │
    │  startLevel(levelId) → 加载 RoomLevel.scene
    │  backToMenu() → 加载 MainMenu.scene
    ▼
LevelController（RoomLevel.scene 根节点脚本）
    │  onLoad: 从 LevelConfig 读取配置，初始化安睡度=100，启动倒计时
    │  update: 每帧检查倒计时和安睡度，判定胜负
    │
    ├──▶ EventScheduler（子节点脚本）
    │       按 eventInterval 定时，从 eventPool 随机选事件
    │       检查并发数 < maxConcurrent，实例化事件预制体
    │       事件预制体挂载到场景中预设的锚点位置
    │
    ├──▶ HUD（子节点脚本）
    │       监听 EventBus.SLEEP_CHANGED → 更新安睡度条
    │       监听 LevelController 的倒计时 → 更新时间文字
    │
    └──▶ ResultPanel（动态实例化的 Prefab）
            通关/失败时由 LevelController 创建
            按钮事件 → GameManager.startLevel(1) 或 backToMenu()

BaseEvent（事件预制体上的脚本基类）
    │  init(eventConfig, responseTimeMultiplier)
    │  启动时间窗口倒计时
    │  子类实现具体交互判定
    │  成功 → EventBus.emit(EVENT_RESOLVED, penalty)
    │  超时 → EventBus.emit(EVENT_FAILED, penalty)
    │  完成后自动销毁节点
    │
    ├── ClickEvent extends BaseEvent
    │     touch_start → 成功
    │
    └── LongPressEvent extends BaseEvent
          touch_start → 开始累计
          touch_end → 重置
          累计 >= requiredDuration → 成功
```

### EventBus 事件定义

| 事件名 | 触发时机 | 携带数据 |
|--------|---------|---------|
| SLEEP_CHANGED | 安睡度变化时 | `{ value: number }` |
| EVENT_SPAWNED | 事件预制体实例化后 | `{ eventId: string }` |
| EVENT_RESOLVED | 玩家成功处理事件 | `{ eventId: string, penalty: number }` |
| EVENT_FAILED | 事件超时 | `{ eventId: string, penalty: number }` |
| LEVEL_END | 关卡结束 | `{ success: boolean, sleepValue: number }` |

### 关键设计决策

- **GameManager 为静态类单例**：不挂载到节点，纯 TypeScript 静态类。只持有 `currentLevelId` 和提供 `startLevel()` / `backToMenu()` 方法（内部调用 `director.loadScene`）。无需 `addPersistRootNode`。
- **事件预制体 = 节点 + 脚本**：每种事件是独立预制体，挂载对应的事件脚本，EventScheduler 只负责实例化和放置
- **不做对象池**：第一关并发最多1个事件，无需池化
- **首个事件出现时机**：关卡开始后等待一个 `eventInterval`（8秒）再生成第一个事件，给玩家视觉缓冲
- **倒计时归零处理**：倒计时归零时立即判定通关（安睡度 > 0），销毁所有活跃事件节点

### 节点约定

- 事件预制体实例化后，由 EventScheduler 设为 EventLayer 的子节点，移动到对应 Anchor 的 position
- Anchor 节点不可见，只是位置标记
- LevelRoot 不在 EventLayer 下，避免父子关系混乱。LevelController 通过引用获取 EventLayer 和 HUD
